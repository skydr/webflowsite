Product

Low-code service builder. Using the design system and separately designed smart elements, it allows you to quickly develop a unique design for services in SberFriend.

Audience

Contractors and external service developers at Sberbank

Product metrics

Service delivery speed. User engagement.

My role

Project control. Participation in backlog planning and component base prioritization.

Designing a service

Wysiwyg web constructor allows you to assemble visual elements and screens for SberFriend.

Smart elements and templates

Ready-made components based on the design system and pre-programmed logic of frequently used cases can reduce development time. There is also a system of templates for frequently used interface solutions.

Cross-platform

The solution allows you to create both adaptive web screens and native mobile screens for the same services. Because a single design system is used.

For products

Uses the SberFriend product design system and publishes services in its catalog.

Code & accesses

Connecting to GIT or editing code to manage the service.

Creating logic

Frontend is completely formed in the constructor and it remains only to supplement the logic and control of the service.

Debugging and Testing

There is a console displaying information about test servers and test tabs for debugging on the required version of the service.

Entry point, keys, directories

There are also ready-made keys, entry points and used directories for use in the service.

Service management

There is everything you need to manage the service by the developer, manager and the whole team.

Collaboration

It is possible to exchange ready-made solutions in the template database. Manage and edit one project simultaneously by several developers.

Verification, publication and versions

The publication process is divided into several stages of verification. Each publication creates a separate version. It is possible to change the published version to any stable approved version.
